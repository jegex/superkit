# Contributing

Thank you for considering contributing to Superkit.

## Bug Reports

Open an issue on [GitHub](https://github.com/jegex/superkit/issues). Include:

- PHP & Laravel version
- Steps to reproduce
- Expected vs actual behavior

## Feature Requests

Open an issue describing the feature, its use case, and how it fits the starter kit vision.

## Pull Requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/my-feature`)
3. Make your changes
4. Run tests: `php artisan test`
5. Run Pint: `vendor/bin/pint --format agent`
6. Push and open a PR against `main`

### Style

- Follow existing code conventions
- Use PHP 8.4 features where appropriate
- Add tests for new functionality
- Keep PRs focused on a single concern
